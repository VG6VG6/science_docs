# Science Docs (Система проверки научных публикаций)

Веб-приложение для автоматизированного поиска и верификации научных статей, агрегирующее данные из **Scopus API** (авторы, названия, журналы, год) с локальной базой метрик **Scimago** (Квартили, SJR, H-index, страны). Решает проблему лимитов API и собирает консолидированные данные в удобном для экспорта виде.

## 🌟 Основные возможности

* **Умный поиск по статьям**: Поиск по названию или DOI статьи с автоматическим обогащением наукометрическими показателями.
* **Полноценный поиск по авторам**: Выгрузка *всех* публикаций автора (с обходом лимита в бесплатной версии Scopus API).
* **Локальная база данных (Warehouse)**: Загружает метрики из CSV-файлов рейтингов Scimago (от 2020 до 2024 года).
* **Кэширование**: Повторные поиски авторов или статей используют локальную БД и не расходуют лимит Scopus API.
* **Пакетная обработка (Batch)**: Python-скрипты для массовой фоновой проверки огромного массива публикаций из файлов.
* **Экспорт в Excel**: Адаптированный под русскоязычные системы экспорт результатов в формате CSV (с поддержкой BOM и разделителем-точкой с запятой).

## 🛠 Стек технологий

* **Backend:** Python + FastAPI + SQLAlchemy + SQLite
* **Frontend:** HTML5 + CSS3 + Vanilla JS
* **Инфраструктура:** Docker + Docker Compose

## ⚙️ Настройка и конфигурация

1. Создайте директорию `bin` в корне проекта (если её нет).
2. Внутри `bin` создайте конфигурационный файл `info.env`.
3. Укажите ваш API-ключ от Scopus:

```env
SCOPUS_API_KEY=ваш_scopus_api_ключ_здесь
```

## 🏗️ Подготовка базы журналов (Scimago)

Для корректного отображения рейтингов (SJR, квартили) требуется импортировать таблицы журналов. Поместите скачанные файлы вида `scimagojr 2020.csv` ... `scimagojr 2024.csv` в корневую директорию проекта и запустите первоначальный импорт через `main.py`:

```bash
# Локально (если установлен Python):
python3 main.py -u -r
```
* Флаг `-u` обновляет/импортирует данные из CSV "journals_ranks".
* Флаг `-r` перестраивает внутреннюю базу "warehouse".
*(База автоматически сохранится в файлы `science_docs.db` / `science_warehouse.db` внутри папки `bin`)*

## 🚀 Запуск через Docker (Рекомендуется)

Проект полностью обёрнут в Docker, что избавляет от необходимости локально настраивать Python и библиотеки. Папка `bin` пробрасывается в контейнер, так что собранная база данных и кэши не пропадут при пересоздании приложения.

```bash
# Билд и запуск в фоновом режиме:
sudo docker compose up --build -d

# Остановка (БД сохранится локально):
sudo docker compose down

# Перезапуск (если что-то сломалось или зависло):
sudo docker compose restart

# Просмотр логов бэкенда в реальном времени:
sudo docker compose logs -f
```

После старта интерфейс приложения будет доступен по адресу:
🌐 **http://localhost:8000** 

Автосгенерированная документация API (Swagger):
📘 **http://localhost:8000/docs**

## 💻 Локальный запуск (Без Docker)

Если вы разрабатываете приложение и хотите запускать его локально:

1. Создайте и активируйте виртуальное окружение:
```bash
python3 -m venv .venv
source .venv/bin/activate
```
2. Установите зависимости:
```bash
pip install -r requirements.txt
```
3. Запустите сервер (FastAPI/Uvicorn):
```bash
uvicorn app.api:app --host 0.0.0.0 --port 8000 --reload --app-dir app
```

## 🚀 Пакетная обработка в терминале

В `main.py` встроена фича Batch-обработки (массовая проверка публикаций) напрямую из терминала (без веб-интерфейса):
```bash
python3 main.py -b "путь/к/исходнику.xlsx" -o "результат/report.json"
```
